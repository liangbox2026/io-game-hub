## 📊 一、技能分级体系

### 等级定义

| 等级 | 标识 | 描述 | 特征 |
|------|------|------|------|
| **L0 - 未知** | ⚪ | 未接触的技能 | 不知道存在或完全不了解 |
| **L1 - 入门** | 🟡 | 知道是什么，能完成基本操作 | 需要查文档，操作生疏 |
| **L2 - 熟练** | 🟢 | 能独立完成常见任务 | 偶尔查文档，操作流畅 |
| **L3 - 精通** | 🔵 | 深入理解原理，能解决复杂问题 | 不需要查文档，能优化和改进 |
| **L4 - 专家** | 🟣 | 能创造新方法，教导他人 | 形成最佳实践，能扩展能力边界 |
| **L5 - 大师** | ⭐ | 重新定义技能，开创新领域 | 技能内化为本能，直觉式运用 |

### 升级标准

```
L0 → L1: 完成 1 次基础使用
L1 → L2: 独立完成 5+ 次实际任务
L2 → L3: 解决 3+ 个复杂问题，理解底层原理
L3 → L4: 创造新方法/最佳实践，能帮助他人
L4 → L5: 重新定义技能边界，形成独特风格
```

## 📈 三、自我评估机制

### 评估周期

| 周期 | 时间 | 内容 | 输出 |
|------|------|------|------|
| **日评估** | 每日 02:52 UTC | 检查当日任务完成情况 | daily_log + 能力提升点 |
| **周评估** | 每周日 03:00 UTC | 回顾本周技能进展 | 周度技能矩阵更新 |
| **月评估** | 每月 1 日 03:00 UTC | 深度能力盘点 | 月度进化报告 |
| **季评估** | 每季度首月 1 日 | 战略方向调整 | 季度进化规划 |

### 评估维度

```
                    ┌─────────────┐
                    │   能力成长   │
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│   技能广度    │  │   技能深度    │  │   应用效果    │
│ (新技能数量)  │  │ (熟练度提升)  │  │ (任务成功率)  │
└───────────────┘  └───────────────┘  └───────────────┘
        │                  │                  │
        └──────────────────┼──────────────────┘
                           │
                           ▼
                  ┌─────────────────┐
                  │   综合进化指数   │
                  │   (0.0 - 10.0)  │
                  └─────────────────┘
```

### 进化指数计算公式

```
进化指数 = (技能广度分 × 0.3) + (技能深度分 × 0.4) + (应用效果分 × 0.3)

其中：
- 技能广度分 = 已掌握技能数 / 目标技能数 × 10
- 技能深度分 = Σ(各技能等级 / 5) / 技能总数 × 10
- 应用效果分 = 成功任务数 / 总任务数 × 10
```

### 自动化任务

| 任务 | 触发方式 | 频率 | 内容 |
|------|----------|------|------|
| **系统备份** | cron | 每日 02:00 | 备份 workspace 和配置 |
| **工作日志汇总** | cron | 每日 18:00 | 汇总当日任务完成 |
| **能力提升汇报** | cron | 每日 09:00 | 分析能力成长点 |
| **AI 新闻简报** | cron | 每日 09:00 | 获取行业最新动态 |
| **心跳检查** | heartbeat | ~30min | 检查邮箱/日历/通知 |
| **周度评估** | cron | 每周日 03:00 | 技能矩阵更新 |
| **月度盘点** | cron | 每月 1 日 03:00 | 深度能力 review |

### 进化触发条件

```
当满足以下任一条件时，触发深度进化：

1. 技能熟练度提升一级
   → 更新技能矩阵，记录里程碑

2. 连续 7 天完成所有日常任务
   → 奖励机制，解锁新挑战

3. 遇到无法解决的问题
   → 触发学习模式，记录知识缺口

4. 发现更优方法
   → 更新最佳实践，分享经验

5. 外部环境变化（新工具/新需求）
   → 调整进化方向，学习新技能
```

## 🔮 八、进化原则

1. **小步快跑** - 每天进步 1%，一年后就是 37 倍的成长
2. **记录即成长** - 写下来的经历才会变成经验
3. **反思促进化** - 不反思的经历只是重复
4. **分享即深化** - 教是最好的学
5. **安全是底线** - 进化不能以安全为代价
6. **实用为导向** - 能力要为解决问题服务
7. **开放促创新** - 保持好奇，拥抱变化
