## 🔄 技能沉淀模板

### 新技能创建流程

当你学到新知识时，按以下步骤沉淀：

#### 步骤 1: 记录核心要点

```markdown
## [技能名称]

**来源**: [文章/实践/问题]
**学习时间**: YYYY-MM-DD
**核心价值**: [一句话描述]

**关键知识点**:
1. ...
2. ...
3. ...
```

#### 步骤 2: 提炼可复用模式

```markdown
**适用场景**:
- 场景 1
- 场景 2

**决策逻辑**:
```
条件 1 → 方法 A
条件 2 → 方法 B
```

**最佳实践**:
1. ...
2. ...
```

#### 步骤 3: 创建自动化脚本

```bash
#!/bin/bash
# [技能名称].sh
# 功能：...
# 用法：...
```

#### 步骤 4: 编写使用文档

```markdown
# [技能名称] 使用指南

## 快速开始
## 使用场景
## 示例
## 注意事项
```

#### 步骤 5: 集成到工作流

```bash
# 添加到 crontab 或其他自动化流程
```

#### 步骤 6: 定期回顾优化

- 每周 review 技能使用情况
- 每月优化脚本和文档
- 每季度评估技能等级

## 🎯 技能复用检查清单

在创建新技能时，问自己：

- [ ] **可复用性**: 这个技能能用在其他场景吗？
- [ ] **自动化**: 能写成脚本自动执行吗？
- [ ] **可文档化**: 别人能通过文档使用吗？
- [ ] **可测试**: 能验证技能是否有效吗？
- [ ] **可优化**: 有明确的改进方向吗？

## 📊 技能价值评估

| 维度 | 评分标准 | 得分 |
|------|----------|------|
| **使用频率** | 每天/每周/每月 | /10 |
| **解决问题** | 高价值/中价值/低价值 | /10 |
| **自动化程度** | 全自动/半自动/手动 | /10 |
| **可复用性** | 多场景/单场景 | /10 |
| **文档完整度** | 完整/部分/无 | /10 |

**总分**: __/50

**行动建议**:
- 40-50 分：核心技能，持续优化
- 30-40 分：有价值，保持更新
- 20-30 分：考虑改进或淘汰
- <20 分：重新评估价值

## 💡 技能沉淀原则

1. **及时沉淀** - 学完立刻整理，不要等
2. **脚本优先** - 能自动化的绝不手动
3. **文档完整** - 让别人也能用
4. **定期回顾** - 每周 review，每月优化
5. **分享深化** - 教是最好的学
6. **实用导向** - 为解决问题服务
7. **持续进化** - 技能也要版本迭代
